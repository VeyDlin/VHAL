# Getting Started

## Required Tools

All tools must be available in your system `PATH`.

### CMake

Build system generator.

| OS | Install |
|----|---------|
| Windows | [cmake.org/download](https://cmake.org/download/) or `winget install Kitware.CMake` |
| Linux | `sudo apt install cmake` (Ubuntu/Debian) or `sudo pacman -S cmake` (Arch) |
| macOS | `brew install cmake` |

Verify: `cmake --version`

### Ninja

Fast build tool used by CMake.

| OS | Install |
|----|---------|
| Windows | [github.com/ninja-build/ninja/releases](https://github.com/ninja-build/ninja/releases) or `winget install Ninja-build.Ninja` |
| Linux | `sudo apt install ninja-build` |
| macOS | `brew install ninja` |

Verify: `ninja --version`

### ARM GCC Toolchain

Cross-compiler for ARM Cortex-M targets.

| OS | Install |
|----|---------|
| Windows | [developer.arm.com/downloads/-/arm-gnu-toolchain-downloads](https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads) — select the `.exe` installer, it adds to PATH automatically |
| Linux | `sudo apt install gcc-arm-none-eabi` |
| macOS | `brew install arm-none-eabi-gcc` |

Verify: `arm-none-eabi-gcc --version`

### OpenOCD

On-chip debugger for flashing and debugging via ST-Link, CMSIS-DAP, J-Link.

| OS | Install |
|----|---------|
| Windows | [github.com/xpack-dev-tools/openocd-xpack/releases](https://github.com/xpack-dev-tools/openocd-xpack/releases) — extract and add `bin/` to PATH. Also bundled with STM32CubeIDE. |
| Linux | `sudo apt install openocd` |
| macOS | `brew install openocd` |

Verify: `openocd --version`

> **Windows tip:** If you have STM32CubeIDE installed, OpenOCD is at:
> `C:\Program Files\ST\STM32CubeIDE_*\STM32CubeIDE\plugins\com.st.stm32cube.ide.mcu.externaltools.openocd.win32_*\tools\bin\`
> Add this path to your system PATH, or install the xpack standalone version.

## Demo Projects

The repository includes ready-to-use demo projects in the `.demo/` directory. Each demo is a complete project with pre-configured CMake presets, VS Code tasks, and debug configurations. You can use any demo as a starting point for your own project — just copy it and modify as needed.

## Project Workflow

### Build

Use the **Build** button in the status bar, or:

```bash
cmake --preset Debug
cmake --build --preset Debug
```

### Flash

Use the **Flash** button in the status bar, or run the Flash task from the command palette (`Ctrl+Shift+P` > `Tasks: Run Task` > `Flash`).

Under the hood this runs OpenOCD:
```bash
# STM32 (ST-Link)
openocd -f interface/stlink.cfg -f target/stm32g0x.cfg -c "program build/Debug/Firmware.elf verify" -c "reset" -c "exit"

# ENS001 (CMSIS-DAP)
openocd -f interface/cmsis-dap.cfg -f target/swj-dp.tcl -c "program build/Debug/Firmware.elf verify" -c "reset" -c "exit"
```

### Debug

Press `F5` or use `Run and Debug` panel. Two configurations are available:

- **Debug (OpenOCD)** — uses ST-Link or CMSIS-DAP
- **Debug (J-Link)** — uses Segger J-Link

The debugger will build, flash, and halt at `main()`.

## Troubleshooting

**`openocd` not found:**
OpenOCD is not in your PATH. Install it and ensure the `bin/` directory is in your system PATH. Restart VS Code after changing PATH.

**`arm-none-eabi-gcc` not found:**
ARM toolchain is not in PATH. On Windows, re-run the installer and check "Add to PATH", or manually add the `bin/` directory.

**ST-Link not detected:**
Install ST-Link drivers. On Windows: [ST-Link USB drivers](https://www.st.com/en/development-tools/stsw-link009.html). On Linux: `sudo apt install stlink-tools` or add udev rules.

**CMSIS-DAP not detected:**
Ensure the debug probe firmware supports CMSIS-DAP. No special drivers needed on modern OS versions.
